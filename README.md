## О проекте

BRS Calculator — локальный калькулятор рейтинга по [Положению о БРС ЮУрГУ](https://edu.susu.ru/handbook/ru/pps/brs). Приложение **не подключается к университету**: все данные вводятся вручную и хранятся только на устройстве пользователя.

> Это не электронный журнал, а инструмент для быстрого ответа:  
> **«Какой у меня рейтинг и что будет, если я получу другие баллы?»**

## Возможности

| Функция | Описание |
| ------- | -------- |
| **Предметы** | Создание дисциплин с названием и типом аттестации |
| **Тип аттестации** | Экзамен (оценки 2–5) или зачёт (зачтено / незачтено) |
| **Контрольные точки** | Название, балл, максимум, вес; дробные значения (`0,15`, `4,5`) |
| **Промежуточная аттестация** | Отдельная точка (экзамен / зачёт) |
| **Бонус-рейтинг** | Поощрения и штрафы по показателям (−15 … +15 %) |
| **Автопересчёт** | R_тек и R_d обновляются мгновенно |
| **Оценка до аттестации** | При R_тек ≥ 60 % показывается зачёт или оценка 3 / 4 / 5 |
| **Что будет если…** | Симуляция баллов для нескольких точек сразу, без сохранения |
| **Сколько нужно на экзамен?** | Reverse calculator — минимальный балл для цели (3 / 4 / 5 / зачёт) |
| **Шаблоны предметов** | Готовые наборы контрольных точек (экзамен, физкультура, лаб. курс) |
| **Визуализация** | Кольцевой gauge R_тек, шкала оценок, анимация count-up |
| **Обзор семестра** | Сводка: предметы в норме, под угрозой, средний рейтинг |
| **Поиск** | Фильтрация списка предметов по названию |
| **Прогресс в списке** | Мини-шкала R_тек у каждого предмета |
| **Дублирование** | Копия предмета со всеми точками и бонусами |
| **Сворачиваемые блоки** | «Сколько нужно» и «Что если» — не перегружают экран |
| **Уведомления** | Toast «Сохранено» и тактильная отдача в Telegram |
| **Переименование** | Редактирование названия предмета без пересоздания |
| **Hash-роутинг** | `#/`, `#/new`, `#/subject/:id` — работает back и deep links |
| **Резервная копия** | Экспорт / импорт всех предметов в JSON |
| **Тема** | Светлая / тёмная, переключатель в шапке; синхронизация с Telegram |

## Формулы расчёта

Все вычисления — в `src/calculator/` строго по п. 3.3 Положения о БРС.

### Рейтинг по мероприятию

```
r_i = (b_i / b_i_max) × 100
```

### Рейтинг по текущему контролю

```
R_тек = Σ(w_i × r_i) / Σ(w_i)
```

Учитываются **все** точки текущего контроля. Незаполненный балл = **0** (как в Univeris).

### Рейтинг по промежуточной аттестации

```
R_pa = (b_pa / b_pa_max) × 100
```

### Бонус-рейтинг

```
R_b = Σ B_j   (ограничен [−15 %; +15 %])
```

### Итоговый рейтинг

```
R_d = 0.6 × R_тек + 0.4 × R_pa + R_b
```

Итог ограничен диапазоном **0–100 %**.

### Отказ от аттестации (п. 3.3)

Если **R_тек ≥ 60 %** и аттестация не сдана:

```
R_d = R_тек + R_b
```

### Шкала оценок (экзамен)

| Оценка | Рейтинг |
| ------ | ------- |
| Отлично (5) | 85–100 % |
| Хорошо (4) | 75–84 % |
| Удовлетворительно (3) | 60–74 % |
| Неудовлетворительно (2) | &lt; 60 % |

Для **зачёта**: ≥ 60 % — зачтено, &lt; 60 % — незачтено.

Округление — до ближайшего целого **вверх** (п. 3.3).

**Отображение оценок в приложении:**
- в списке предметов и сводке — оценка по итоговому `displayRating` (R_d, R_тек+R_b или R_тек);
- в карточке предмета под R_тек — оценка только по текущему контролю;
- под блоком R_d — оценка по соответствующему итоговому значению.

## Примеры расчёта

### Экзамен

| Точка | Балл | Макс. | Вес |
| ----- | ---- | ----- | --- |
| Практическая №1 | 4 | 5 | 1 |
| Практическая №2 | 5 | 5 | 1 |
| Лабораторная №1 | 9 | 10 | 3 |
| Экзамен | 40 | 50 | — |
| Бонус (конференция) | +2 % | | |

```
R_тек = (80×1 + 100×1 + 90×3) / 5 = 90 %
R_pa  = 40/50 × 100 = 80 %
R_b   = +2 %
R_d   = 0.6×90 + 0.4×80 + 2 = 88 %  →  хорошо (4)
```

### Физическая культура (зачёт)

| Точка | Балл | Макс. | Вес |
| ----- | ---- | ----- | --- |
| Задания (фев–март) | 50 | 80 | 22 |
| Задания (апр–май) | 70 | 75 | 38 |
| Занятия | 0 | 5 | 15 |
| Нормативы | 1 | 5 | 15 |
| Теория | 10 | 10 | 10 |

```
R_тек = 62.22 %
R_d   = 62.22 %  (без аттестации, R_тек ≥ 60 %)
       →  зачтено
```

## Быстрый старт

```bash
git clone <repo-url>
cd brs
npm install
npm run dev      # http://localhost:5173
```

С телефона в той же Wi‑Fi сети: адрес вида `http://192.168.x.x:5173` (показывается в терминале).

```bash
npm run build    # production-сборка
npm run preview  # http://localhost:4173
npm test         # 32 unit-теста
npm run lint     # oxlint
npm run check    # lint + test + build
```

> **Важно:** не открывайте `index.html` или файлы из `dist/` двойным щелчком — приложение работает через dev-сервер или `npm run preview`.

### Если «страница недоступна»

1. Выполните `npm install`
2. Запустите `npm run dev` и дождитесь `ready`
3. Откройте **http://localhost:5173**, а не файл на диске
4. Проверьте, что порт 5173 не занят
5. Для production: `npm run build && npm run preview`

## Хранение данных

Данные хранятся локально в **IndexedDB** через Dexie.js:

```typescript
subjects: [{
  id,
  name,
  disciplineType: 'exam' | 'credit',
  assessments: [{ id, name, score, maxScore, weight, type }],
  bonuses: [{ id, name, value }]
}]
```

- Без регистрации и авторизации
- Без серверной синхронизации
- Данные сохраняются после закрытия Telegram
- **Резервная копия:** на главном экране → «Резервная копия» → экспорт JSON; импорт заменяет или дополняет данные

## Деплой на Vercel

Проект уже содержит `vercel.json`, `.nvmrc` (Node 20) и `.npmrc` — **дополнительные настройки в панели Vercel не нужны**.

1. Подключите репозиторий к [Vercel](https://vercel.com)
2. Убедитесь, что **Framework Preset** = **Other** (или Vite — `vercel.json` переопределит команды)
3. **Root Directory** — корень репозитория (где `package.json`)
4. Push в `main` → Vercel соберёт `dist/` автоматически
5. URL в Telegram Bot → Menu Button / Web App

`vercel.json` задаёт:

| Параметр | Значение |
| -------- | -------- |
| Install | `npm ci --include=dev` |
| Build | `npm run build` |
| Output | `dist` |

> **Важно:** не задавайте переменную окружения `NODE_ENV=production` в Vercel **до сборки** — без devDependencies не установятся `vite` и `typescript`, и билд упадёт с `'tsc' is not recognized`. Файл `.npmrc` и `installCommand` в проекте это страхуют.

### Если деплой падает

| Ошибка в логе | Решение |
| ------------- | ------- |
| `'tsc' / 'vite' is not recognized` | Redeploy после push с актуальным `vercel.json`; проверьте, что `NODE_ENV` не блокирует devDeps |
| `'npm ci' … not in sync` / `Missing: @emnapi/core` | Закоммитьте актуальный `package-lock.json` (`npm install` локально → push) |
| `Could not resolve "./src/main.tsx"` | Убедитесь, что деплоится последний commit (не старый кэш) |
| Белый экран в Telegram | Свежий deploy + полностью закройте Mini App (см. ниже) |

Навигация в приложении — **hash-роутинг** (`#/`, `#/new`, `#/subject/:id`), rewrites не ломают `/assets/*`.

## Telegram Mini App

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Настройте Web App URL на ваш деплой (**обязательно HTTPS**)
3. Откройте приложение через кнопку меню бота

Приложение подстраивается под тему Telegram. Ручной переключатель темы в шапке сохраняет выбор пользователя.

В `index.html` подключён официальный скрипт `telegram-web-app.js` — это нужно для корректной работы внутри Telegram WebView.

### Белый экран в Telegram

**Главная причина:** Vite добавляет атрибут `crossorigin` к `<script>` и `<link>`. WebView Telegram блокирует такие файлы — JS не загружается, экран остаётся белым. В проекте это исправлено плагином `telegramCompatPlugin` в `vite.config.ts`.

Если в браузере работает, а в Telegram — нет:

1. **Задеплойте свежую сборку** (`npm run build` → push на Vercel)
2. **Полностью закройте** Mini App и откройте снова (Telegram кэширует страницы)
3. URL в BotFather: `https://ваш-домен.vercel.app/` (**HTTPS**)
4. После обновления вы должны увидеть «**Загрузка…**», затем приложение. Если ошибка — появится текст с причиной.
